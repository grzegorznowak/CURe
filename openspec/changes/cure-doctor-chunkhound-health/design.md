## Module: `cure_chunkhound.py`

New module at project root (flat module layout). Contains reusable ChunkHound MCP/JSON-RPC
logic extracted from the generated helper script in `cure_llm.py`.

### Source of extracted logic

Currently in `cure_llm.py`, `write_chunkhound_helper()` (line 1733) generates a Python
script string containing inline definitions of `JsonRpcSession` (line 2052), `_base_cmd()`
(line 1951), `_run_preflight()` (line 2499), `_run_tool_once()` (line 2790), and
`_daemon_metadata_payload()` (line 1975). These are NOT top-level functions — they
are string content.

The extraction moves this logic into `cure_chunkhound.py` as real top-level functions.
`write_chunkhound_helper()` is refactored to generate a thin wrapper script that
`import`s from `cure_chunkhound` at runtime (the helper runs in the same venv where
CURe is installed).

## Doctor Check: `chunkhound-config-validate`

### Logic

```python
# Canonical recommendation table (single source of truth)
_RECOMMENDED = {
    "embedding": {
        "provider": "voyageai",
        "model": "voyage-3.5-lite",
        "rerank_model": "rerank-2.5",
        "api_key": ...,  # non-empty check
    },
    "llm": {
        "provider": "deepseek",
        "base_url": "https://api.deepseek.com",
        "synthesis_model": "deepseek-v4-flash",
        "utility_model": "deepseek-v4-flash",
        "api_key": ...,  # non-empty + starts with sk-
        "codex_reasoning_effort_synthesis": "high",
        "codex_reasoning_effort_utility": "high",
    },
}

def _validate_chunkhound_config(config: dict) -> tuple[str, str]:
    # Hard fail: required sections
    if "embedding" not in config:
        return ("fail", "missing required section: embedding")
    if "llm" not in config:
        return ("fail", "missing required section: llm")

    # Soft warn: deviations from recommendation
    warnings = []
    for section in ("embedding", "llm"):
        actual = config.get(section, {})
        for field, expected in _RECOMMENDED[section].items():
            value = actual.get(field)
            if field == "api_key":
                if not value:
                    warnings.append(f"{section}.api_key is empty")
                elif section == "llm" and not str(value).startswith("sk-"):
                    warnings.append(f"{section}.api_key does not start with sk-")
            elif value != expected:
                warnings.append(
                    f"recommended {section}.{field} is {expected!r}, got {value!r}"
                )

    if warnings:
        return ("warn", "; ".join(warnings))
    return ("ok", "configuration matches CURe recommendation")
```

## Doctor Check: `chunkhound-health`

### Flow

```text
1. Load user's validated ChunkHound config (embedding + llm)
2. Check credentials: `embedding.api_key` + `llm.api_key` in resolved config
   ↓ config credentials missing → DoctorCheck(status="warn", "skipping runtime check...")
   ↓ config credentials available
3. index_fixture_for_health_check(chunkhound_binary, user_config, timeout=120)
   ↓ (merged_config_path, repo_path, temp_dir)
4. run_chunkhound_mcp_preflight(merged_config_path, repo_path, timeout=30)
   ↓ ChunkHoundPreflightResult
5. if preflight failed → DoctorCheck(status="fail", ...)
   ↓ preflight OK
6. run_chunkhound_tool(merged_config_path, repo_path, "search",
       {"type": "regex", "query": "def saludar"}, timeout=60)
   ↓ search result — must reference main.py or utils.py
7. if search returned no fixture results → DoctorCheck(status="fail", ...)
   ↓ search OK
8. run_chunkhound_tool(merged_config_path, repo_path, "code_research",
       {"query": "como funciona la funcion saludar"}, timeout=300)
   ↓ research result — must contain at least one citation to a fixture file
9. DoctorCheck(status="ok", ...)
   or on research timeout → DoctorCheck(status="warn", ...)
   or on research without citation → DoctorCheck(status="warn", ...)
```

### Timeout handling

- Credential check: instantaneous (resolved config lookup)
- Fixture indexing: 120s
- Preflight: 30s (spawn + initialize)
- Search: 60s (single regex query)
- Code research: 300s (can be slow)
- Total worst case: ~510s (~8.5 min)

### Status mapping

| Condition | Status |
|---|---|
| Resolved config lacks `embedding.api_key` or `llm.api_key` | `warn` |
| Fixture indexing fails | `fail` |
| Fixture indexing timeout | `warn` |
| Preflight fails | `fail` |
| Preflight OK, search returns no fixture results | `fail` |
| Preflight OK, search OK, research OK with citation | `ok` |
| Preflight OK, search OK, research timeout | `warn` |
| Preflight OK, search OK, research returns no citation | `warn` |

### Search and research success criteria

- **Search success**: result contains at least one file path referencing `main.py`
  or `utils.py` from the fixture
- **Research success**: response contains at least one citation (markdown link or
  file path reference) to a fixture file (`main.py`, `utils.py`, or `README.md`)

### Gate conditions

`chunkhound-health` is only appended to the check list when ALL of:
1. `chunkhound-config` check status is `ok` (existing check #4 — base config loadable)
2. `chunkhound` binary check status is `ok` (existing check #10)
3. `chunkhound-config-validate` status is NOT `fail` (warn is ok — soft recommendations
   don't block runtime verification)

Rationale: `chunkhound-config` only validates that `base_config_path` points to readable
JSON. It does NOT verify that `embedding` or `llm` sections exist within that JSON. If
they are missing, MCP preflight will fail, so we gate health out early.

### Shared artifacts

```python
# cure_runtime.py
def _doctor_runtime_checks(runtime, *, artifacts: dict | None = None, ...):
    ...
    health_check, preflight_result = _doctor_chunkhound_health_check(runtime)
    checks.append(health_check)
    if artifacts is not None and preflight_result is not None:
        artifacts["chunkhound_health"] = preflight_result

def _doctor_runtime_payload(runtime, *, artifacts: dict | None = None, ...):
    ...
    if artifacts and "chunkhound_health" in artifacts:
        result = artifacts["chunkhound_health"]
        payload["chunkhound_health"] = {...}
```

## Shared Artifacts Data Flow

```text
cure_commands.doctor_flow():
    artifacts = {}
    checks = _doctor_runtime_checks(runtime, ..., artifacts=artifacts)
    if json_output:
        payload = _doctor_runtime_payload(runtime, ..., artifacts=artifacts)
```

`_doctor_runtime_checks()` populates `artifacts` during check execution.
`_doctor_runtime_payload()` reads from it. No re-execution of expensive operations.

## Test Architecture

New file `tests/test_doctor_chunkhound.py`:

1. **Dataclass tests (TAP-1)** — construction, defaults, serialization
2. **Config validation tests (TAP-2)** — parametrized config dicts → expected status,
   covering all fields in the canonical recommendation table
3. **Health status mapping tests (TAP-3)** — mocked preflight/search/research outcomes
4. **Credential check tests (TAP-4)** — resolved-config credential detection
5. **Orchestration skip tests (TAP-5)** — binary/config gate logic
6. **JSON payload tests (TAP-6)** — mocked preflight → expected JSON, absent when skipped
7. **Fixture helper tests (TAP-7)** — mocked subprocess, temp dir, config merge
8. **Migrated tests (TAP-9)** — from `_reviewflow_unittest_runtime_ui_impl.py`

All unit tests use in-memory or mocked fixtures; no real ChunkHound binary needed.
